# SETUP-ANALYTICS — 計測の入れ方（読んで、そのままやるだけ）

> 現状：計測タグは「IDが設定されていれば自動で入る」状態。**まだ ID が無いので計測は動いていない。**
> あなたがやるのは「ID を取る」ことだけ。取った ID を Claude Code に貼れば、こちらで反映して計測が始まる。
> 注意：計測は ID 設定後からスタート（過去に遡ってのデータは取れない）。

---

## A. Google アナリティクス 4（GA4）— 必須

**取得する場所：** https://analytics.google.com/

1. 上のURLにアクセスし、使いたい Google アカウントでログイン。
2. 左下の **歯車「管理」** をクリック → **「+ アカウントを作成」**。
   - アカウント名：例 `便利ツール集` → 次へ。
3. **「プロパティを作成」**：
   - プロパティ名：例 `benri-tools`
   - レポートのタイムゾーン：**日本**
   - 通貨：**日本円（JPY）**
   - → 次へ → ビジネス情報を適当に選択 → **「作成」** → 利用規約に同意。
4. **データストリームを作成**：プラットフォームで **「ウェブ」** を選ぶ。
   - ウェブサイトのURL：あなたの公開URL（例 `https://xxxx.pages.dev`）
   - ストリーム名：例 `web`
   - → **「ストリームを作成」**。
5. 表示された画面の上部に **「測定ID」** がある。形は **`G-XXXXXXXXXX`**。これを **コピー**。

**→ Claude Code に「GA4の測定IDは `G-XXXXXXXXXX`」と伝えるだけ。**
（こちらで Cloudflare の環境変数 `NEXT_PUBLIC_GA4_ID` に設定 → 再デプロイ → 計測開始。）

---

## B. Google Search Console（検索流入・インデックス確認）— 推奨

**取得する場所：** https://search.google.com/search-console

1. 上のURLにアクセス → **「プロパティを追加」**。
2. **「URL プレフィックス」** の方に、公開URL（例 `https://xxxx.pages.dev/`）を入力 → 続行。
3. **所有権の確認**：
   - 先に A の GA4 を入れてあれば、確認方法の **「Google アナリティクス」** でワンクリック確認できることが多い。
   - できない場合は **「HTML タグ」** 方式を選び、表示される `<meta name="google-site-verification" content="……">` の **content の値** を Claude Code に伝える（こちらで埋め込む → あなたが「確認」を押す）。
4. 確認できたら、左メニュー **「サイトマップ」** → 「新しいサイトマップの追加」に **`sitemap.xml`** と入力して送信。

**→ Claude Code に伝えるもの：** 登録したプロパティURL（と、HTMLタグ確認を使った場合はその content 値）だけ。

---

## C. まとめ：あなたが Claude Code に渡すもの
| 項目 | 形 | どこで取る |
|---|---|---|
| GA4 測定ID | `G-XXXXXXXXXX` | A-5 |
| （任意）Search Console 所有権確認の content | 文字列 | B-3（HTMLタグ方式の場合のみ） |

これだけ渡せば、計測の有効化・サイトマップ送信はこちらで進めます。
※ GA4 の数値を Claude が自動で読みに行く運用（月次レポート等）は後段。その時に API 資格情報を GitHub Secrets に入れる手順を別途出します。
