# HUMAN-SETUP — オーナーが一度だけやること（最小）

> 方針：あなたの手作業は極小化。コードでできることは全部エージェントがやる。
> ただし「アカウント作成・本人/税/支払い情報・規約同意・審査承認」は**あなた本人**にしかできない（安全上もそうあるべき）。
> 下記が"避けられない最小リスト"。済んだら各IDをエージェントに伝えるだけ。

## A. 一度だけやること
1. **GitHub アカウント＋空リポジトリ**作成（コード置き場）。無料。
2. **Cloudflare アカウント**作成 → Pages にこのリポジトリを接続。無料・クレカ不要。
3. （AdSenseを使うなら）**独自ドメインを1つ取得**（年¥1,000〜2,000）。DNS を Cloudflare に向ける。
   - ※ホスティング自体は `*.pages.dev` の無料サブドメインでも可。だが **AdSense は所有ドメインが必須**（無料サブドメインは審査を通せない）。ドメイン代は実質「AdSense の入場料」。
   - ※完全無料で始めるなら：無料サブドメイン＋**アフィリだけ**で公開 → 当たったツールにだけドメイン購入 → AdSense 追加。
4. **アフィリ/ASP登録**（使う分だけ）：A8.net / もしも / バリューコマース / Amazonアソシエイト / 楽天 等。本人確認・口座・規約同意。
5. **Google AdSense登録**（トラフィックが付いてから）：本人/住所/税/支払い→ポリシー同意→サイト追加＆審査。
   - 審査は「実在する中身＋本物のアクセス＋必須ページ(About/Contact/Privacy/Terms)」が前提。エージェントが必須ページと枠は用意済みにする。
6. **GA4 と Search Console**：プロパティ作成 → ドメイン所有権の"確認"クリック → AdSense↔GA4 / GSC↔GA4 を連携（ページ別の閲覧数・収益が取れる）。

## B. エージェントに渡すもの（IDは秘密情報ではないので会話で伝えてOK）
- GA4 測定ID： `G-XXXXXXX`
- AdSense 発行者ID： `ca-pub-XXXXXXXXXXXXXXXX`
- Search Console のプロパティ（ドメイン）
- （読み取りAPIを使う場合）API資格情報は**あなたが発行**し、**CIのSecrets**に保存（コードに直書きしない）。エージェントには「Secrets に入れた」とだけ伝える。

## C. 日々やること
- 夜に Claude Code の要約を見て、プレビューを確認し、**マージ/公開を承認**するだけ。

## やらないこと（エージェントに任せる）
広告コードの設置・ads.txt・タグ埋め込み・配置最適化・schema・内部リンク・sitemap・ビルド・デプロイ・計測レポート・改善提案。
