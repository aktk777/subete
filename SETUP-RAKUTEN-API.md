# SETUP-RAKUTEN-API — 楽天APIで「ツール×特定商品」を自動化する（読んで、そのままやるだけ）

> 目的：将来、ツールごとに楽天の**特定商品/カテゴリのアフィリエイトリンク**を自動生成できるようにする。
> あなたがやるのは「IDを2つ取って、置き場所に入れる」だけ。あとはこちらでコード化する。
> 設計ルール：取得は**ビルド時のみ**（生成後は静的リンク）。アプリ実行時に外部APIは叩かない（決定論ルール）。

---

## A. 楽天ウェブサービスの「アプリID（applicationId）」を取る
**場所：** https://webservice.rakuten.co.jp/

1. 上のURLにアクセス → 右上 **「ログイン」**（楽天会員でログイン。無ければ無料登録）。
2. **「アプリID発行」**（=アプリ情報の登録）へ進む。
3. 入力：
   - アプリ名：例 `便利ツール集`
   - アプリURL：あなたの公開URL（例 `https://xxxx.pages.dev`）
   - → 規約に同意して **「アプリを作成」**。
4. 表示された **`applicationId`（数字の長い列）** を控える。

## B. 楽天アフィリエイトID（affiliateID）を取る
**場所：** https://affiliate.rakuten.co.jp/

1. 上のURLにアクセス → 楽天会員でログイン（未登録なら登録）。
2. 発行される **アフィリエイトID** を控える。
   - ※ A の楽天ウェブサービスのアプリ情報画面にも「affiliateId」欄があることが多い。どちらでも同じ値。

## C. 渡し方（秘密として扱う／コードに直書きしない）
`applicationId` は実質APIキー。**GitHub の Secrets** に入れて、私には「入れた」とだけ伝えてください。
1. GitHub リポジトリ → **Settings → Secrets and variables → Actions → New repository secret**
2. 次の2つを登録：
   - `RAKUTEN_APP_ID` ＝ A の applicationId
   - `RAKUTEN_AFFILIATE_ID` ＝ B のアフィリエイトID
3. Claude Code に「楽天のSecretsを入れた」と伝える。
   - ※ Cloudflare のビルド環境変数（同名）に入れる方法でもOK。どちらか一方で良い。

## D. これで実現できること（私が組む）
- 各ツールの `meta.json` に `rakuten` 指定を足せるようにする：
  - キーワード連動：`"rakuten": { "keyword": "ビジネス電卓" }` … 関連商品を自動取得してカード/リンク化
  - 特定商品ピンポイント：`"rakuten": { "itemCode": "shop:item-id" }` … その商品だけに紐付け
- **ビルド時**に楽天商品検索APIで取得 → アフィリエイトID付きの静的リンク/商品カードを生成。
- 結果：ツールごとに「このツールにはこの商品」を結びつけた導線が、量産フローの中で自動生成される。

**あなたの作業は A・B・C の初回だけ。** 以後はツール作成時に商品キーワード/商品コードを指定すれば、リンク生成は自動です。
（Amazonアソシエイトも同様の仕組みが可能ですが、PA-APIは「先に数件の売上」が必要なため、まず楽天で回すのが現実的です。）
